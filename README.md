# hotpepper-sheet ブログ記事の追加方法

このサイトは、GitHub Pagesで動くホットペッパー用ブログ本文案の管理サイトです。現在のブログ一覧ページは `blogs.html` で、個別ページは `article.html?id=記事番号` という形で自動表示されます。

今後は、記事のHTMLを直接編集しません。新しい記事を追加したいときは、**`data/articles.js` に `title` と `content` だけの記事データを一番下へ追加**してください。これだけで、一覧ページ、個別ページ、検索結果に自動で反映されます。

## ファイルの役割

| ファイル | 役割 | 普段触るか |
|---|---|---|
| `data/articles.js` | ブログ記事のデータ置き場です。`title` と `content` だけで管理します。 | **ここだけ触ります** |
| `blogs.html` | 記事一覧ページです。検索機能もここにあります。 | 触りません |
| `article.html` | 個別記事ページです。記事番号に合わせて自動表示されます。 | 触りません |
| `index.html` など | 既存のトップページや他ページです。 | 必要な時だけ触ります |

## 新しい記事を追加する方法

まず、GitHubで `data/articles.js` を開きます。画面右上付近の鉛筆マーク、または `Edit this file` を押して編集画面に入ります。

次に、ファイルの一番下まで移動します。最後の方に、次のような形で記事が並んでいます。

```js
window.BLOG_ARTICLES = [
  {
    title: "【西】梅雨前に朝ラク髪へ",
    content: "こんにちは！西です。\n\n本文が入ります。"
  }
];
```

新しい記事は、最後の記事の後ろに追加します。最後の記事の `}` の後ろに **カンマ `,`** を入れてから、ChatGPTが作った新しい記事データを貼り付けます。

```js
  {
    title: "【西】既存の記事タイトル",
    content: "既存の記事本文"
  },
  {
    title: "【西】新しい記事タイトル",
    content: "こんにちは！西です。\n\nここに新しい本文を入れます。\n\n西"
  }
];
```

保存するときは、GitHub画面下の `Commit changes` を押します。数十秒から数分ほど待つと、GitHub Pagesの `blogs.html` に新しい記事が反映されます。

## ChatGPTに作ってもらう記事データの形式

ChatGPTには、次の形式で出してもらってください。余計な項目は入れず、**title と content だけ**にします。

```js
{
  title: "【西】ここにタイトル",
  content: "こんにちは！西です。\n\nここに本文。\n\n西"
}
```

本文内で改行したい場所には `\n` を使います。空行を入れたい場合は `\n\n` と書きます。

## 追加時に壊さないための注意点

記事追加で触る場所は、基本的に `data/articles.js` だけです。`blogs.html` や `article.html` を編集しなければ、一覧ページや個別ページの仕組みを壊しにくくなります。

| 注意点 | 理由 |
|---|---|
| 最後の記事の後ろにカンマ `,` を入れる | カンマがないと新しい記事とつながらず、表示が止まることがあります。 |
| `title` と `content` 以外を追加しない | サイト側はこの2つだけを読む前提で作っています。 |
| 引用符 `"` を消さない | JavaScriptのデータとして読めなくなります。 |
| `blogs.html` と `article.html` は触らない | 自動生成の仕組みを守るためです。 |
| 追加前に既存記事を消さない | 既存記事も同じデータファイルに入っているためです。 |

## 表示確認の方法

保存後、次のページを開いてください。

<https://nissygram-svg.github.io/hotpepper-sheet/blogs.html>

一覧に新しい記事が増えていれば成功です。検索欄にタイトルの一部を入れて表示されるかも確認してください。個別ページは、一覧の `個別ページ` ボタンから開けます。

## 今後の運用ルール

今後は、ChatGPTが記事データを作り、Manusは必要なときに `data/articles.js` へ追加するだけの運用にできます。サイトの見た目や検索機能はすでに自動化しているため、記事を100本以上に増やす場合も、HTMLを毎回作り直す必要はありません。
