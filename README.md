# hotpepper-sheet ブログ記事管理サイト

このサイトは、GitHub Pagesで動くホットペッパー用ブログ本文案の管理サイトです。ブログ一覧ページは `blogs.html`、個別ページは `article.html?id=記事番号` で表示されます。

今後は、記事のHTMLを直接編集しません。新しい記事を追加したいときは、**`data/articles.js` の追加用ブロックに、ChatGPTが作った `title` と `content` だけの記事データを貼るだけ**です。これだけで、一覧ページ、検索結果、個別ページに自動で反映されます。

## ファイルの役割

| ファイル | 役割 | 普段触るか |
|---|---|---|
| `data/articles.js` | ブログ記事のデータ置き場です。`title` と `content` だけで管理します。 | **ここだけ触ります** |
| `blogs.html` | 記事一覧ページです。検索機能もここにあります。 | 触りません |
| `article.html` | 個別記事ページです。記事番号に合わせて自動表示されます。 | 触りません |
| `README.md` | この記事追加手順を書いている説明書です。 | 困った時だけ見ます |
| `index.html` など | 既存のトップページや他ページです。 | 必要な時だけ触ります |

## 新しい記事を追加する方法

まず、GitHubで `data/articles.js` を開きます。画面右上付近の鉛筆マーク、または `Edit this file` を押して編集画面に入ります。

次に、ファイルの一番下あたりまで移動し、次の目印を探してください。

```js
// ===== ADD NEW ARTICLES BELOW =====
```

この目印の**すぐ下**に、ChatGPTから出力された記事データを貼り付けます。貼り付ける形は、必ず次のようにします。

```js
{
  title: "【西】ここにタイトル",
  content: "こんにちは！西です。\n\nここに本文を入れます。\n\n西"
},
```

複数の記事を一度に追加する場合は、次のように続けて貼り付けてください。

```js
// ===== ADD NEW ARTICLES BELOW =====
{
  title: "【西】新しい記事タイトル1",
  content: "こんにちは！西です。\n\nここに本文を入れます。\n\n西"
},
{
  title: "【西】新しい記事タイトル2",
  content: "こんにちは！西です。\n\nここに本文を入れます。\n\n西"
},
```

貼り付けが終わったら、GitHub画面下の `Commit changes` を押します。数十秒から数分ほど待つと、GitHub Pagesの `blogs.html` に新しい記事が反映されます。

## ChatGPTに作ってもらう記事データの形式

ChatGPTには、次の形式で出してもらってください。余計な項目は入れず、**title と content だけ**にします。

```js
{
  title: "【西】ここにタイトル",
  content: "こんにちは！西です。\n\nここに本文。\n\n西"
},
```

本文内で改行したい場所には `\n` を使います。空行を入れたい場合は `\n\n` と書きます。

## 追加時に壊さないための注意点

記事追加で触る場所は、基本的に `data/articles.js` だけです。`blogs.html` や `article.html` を編集しなければ、一覧ページや個別ページの仕組みを壊しにくくなります。

| 注意点 | 理由 |
|---|---|
| `// ===== ADD NEW ARTICLES BELOW =====` の下に貼る | 追加場所を固定することで、既存記事を誤って消しにくくなります。 |
| 既存記事を編集・削除しない | 既存記事も同じ `data/articles.js` に入っているためです。 |
| 1記事ごとに最後の `},` まで貼る | カンマがないと次の記事とつながらず、表示が止まることがあります。 |
| `title` と `content` 以外を追加しない | サイト側はこの2つだけを読む前提で作っています。 |
| 引用符 `"` を消さない | JavaScriptのデータとして読めなくなります。 |
| `blogs.html` と `article.html` は触らない | 一覧、検索、個別ページの自動表示の仕組みを守るためです。 |

## 表示確認の方法

保存後、次のページを開いてください。

<https://nissygram-svg.github.io/hotpepper-sheet/blogs.html>

一覧に新しい記事が増えていれば成功です。検索欄にタイトルの一部を入れて表示されるかも確認してください。個別ページは、一覧の `個別ページ` ボタンから開けます。

## 今後の運用ルール

今後は、ChatGPTが記事データを作り、Manusは必要なときに `data/articles.js` の追加用ブロックへ貼り付けるだけの運用にできます。サイトの見た目、一覧ページ、検索機能、個別ページは自動で反映されるため、記事を100本以上に増やす場合も、HTMLを毎回作り直す必要はありません。

この運用にすると、Manus側では記事生成やサイト再構築を毎回行わず、**記事データの追加作業だけ**で済むため、クレジット消費を最小化しやすくなります。
